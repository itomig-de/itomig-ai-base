# Contributing to itomig-ai-base

We welcome contributions to itomig-ai-base! Whether you're fixing bugs, improving documentation, or proposing new features, your help is appreciated.

## Reporting Bugs

When reporting a bug, please create an issue on [GitHub](https://github.com/itomig-de/itomig-ai-base/issues) and include:
- **Expected behavior**: What should happen
- **Actual behavior**: What actually happens
- **Steps to reproduce**: How to trigger the issue
- **Proposed fix** (optional): Include code suggestions if you have them

## Contributing Code

To contribute code:

1. **Create a feature branch**: Always work on a dedicated branch (e.g., `feature/fix-anthropic-timeout` or `bugfix/issue-123`)
2. **One PR per change**: Submit separate pull requests for each bug fix or feature
3. **Follow iTop conventions**:
   - Follow the [iTop Coding Standards](https://www.itophub.io/wiki/page?id=latest%3Acustomization%3Acoding_standards)
   - Add translations to dictionary files (minimum: DE, EN, FR)
   - Use the `IssueLog` class for logging
   - Comment your code clearly
   - Write tests covering your changes whenever possible (PHPUnit)

## Code Review Process

We review all contributions carefully but cannot guarantee if, when, or how much of your contribution will be merged into the extension. This project is developed in collaboration with [Combodo](https://github.com/Combodo), and we maintain quality and architectural standards.
